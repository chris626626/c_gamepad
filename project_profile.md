## 核心叙事与行业定位

### 行业共识：DeFi 正在进入"运行阶段"
- DeFi 的下一个竞争维度，不再是"谁上线更快"，而是谁能稳定、长期地运行
- AI/Agent 已进入交易、风控、清算环节
- DeFi 正从"静态合约"走向"持续运行系统"
- 长期运行 = 算力 x 调度 x 执行稳定性

### 市场痛点（DeFi 面临的 4 个结构性瓶颈）
1. **运行成本不可控**：算力调用零散、短期、价格不稳定
2. **执行能力碎片化**：交易、风控、清算分散在不同系统
3. **AI/Agent 无法长期运行**：无状态、无会话、不可持续
4. **缺乏标准化运行层**：每个协议都在"重复造轮子"

### GamePad 做什么
- 为链上系统提供可持续运行的执行与算力底座
- 不要求修改合约、不绑定单一协议、即插即用的运行能力
- 核心理念：Wb3.0 = 新基础设施（不是新协议）

## 整体架构（四层金字塔）

| 层级 | 功能 |
|------|------|
| **应用层** | DEX / 策略 / Agent |
| **执行层** | 交易、风控、清算 |
| **调度层** | 资源与优先级管理 |
| **资源层** | GPU / CPU / 节点网络 |

> 让复杂系统，像云服务一样可调用。

### 系统层级（运行分层逻辑）

| 算力与资源层 | 智能执行层 | 协议运行接口层 |
|-------------|----------|---------------|
| 统一调度 GPU/CPU 资源 | 会话化运行 AI/Agent | 标准化接口接入多类 DeFi 协议 |
| 覆盖交易、清算与 AI/Agent 执行 | 状态保持，支持连续决策 | 无需改动合约即可使用 |
| 提供稳定吞吐与低延迟保障 | 执行过程可控、可追溯 | 执行结果与资源状态双向反馈 |

> 底层算力持续供给，中层智能长期运行，上层协议按需调用。

### 跨层运行能力

| 能力 | 说明 |
|------|------|
| 01 状态与事件闭环 | 全执行路径可追踪，支持状态回溯与执行复盘 |
| 02 模型与策略演进 | 策略版本化管理，灰度升级与快速回滚，支持长期演进 |
| 03 运行可观测与调控 | 执行路径与资源消耗可视化，支持限流、降级与异常控制 |
| 04 隔离与安全边界 | 会话与资源隔离，权限与凭证统一管理，降低系统级风险扩散 |

> 这使 GamePad 更接近"云级运行系统"，而非单一协议。

### 资源调度体系

| 统一资源抽象 | 区分实时与常驻负载 | 状态驱动动态分配 | 长期运行目标 |
|------------|---------------|---------------|------------|
| 交易/风控/Agent 统一为可调度资源单元 | 关键交易与清算优先执行，策略与 Agent 支持长期运行 | 资源随交易活跃度与会话规模自动调整 | 稳定性、弹性扩展、成本可控 |

> 执行能力随系统状态动态变化，而非固定消耗。

### 系统角色（协作型基础设施网络）

| 节点 | 功能 |
|------|------|
| 01 计算节点 | 提供 GPU/CPU 算力资源 |
| 02 协调节点 | 任务调度、资源分配、系统状态维护 |
| 03 执行节点 | AI/Agent 会话管理核心单元，负责执行与状态维护 |
| 04 验证节点 | 关键执行结果验证，支持审计与可信运行 |

## 白名单体系

### 什么是白名单？
GamePad 白名单是参与 GamePad 运行体系的基础设施级身份。代表进入系统运行层、参与长期价值分配与生态演进的资格。

| 项目 | 详情 |
|------|------|
| 全球限量 | 3,000 席 |
| 白名单额度 | 5,000 GPAD |
| 代币名称 | GPAD |
| 代币总量 | 1,000,000,000（固定上限） |
| 释放方式 | 10 期线性释放 |

### 白名单核心权益
1. **运行期价值参与权**：参与系统运行期激励分配，共享执行、算力、Agent 长期运行所产生的系统价值
2. **生态优先参与权**：优先参与核心生态模块与执行网络扩展，早期接入新运行场景、新策略与新 Agent 能力
3. **治理与参数参与权**：参与关键运行参数、激励模型与系统升级方向讨论，对系统长期演进拥有持续话语权
4. **白名单身份与长期绑定**：身份永久记录，作为基础设施早期建设者进入核心生态层

> 白名单不是买入收益，而是进入系统、参与长期运行的资格。

## 核心机制设计理念

1. **生态凭证入场**：进场生态机枪池需要用 10% 的 GPAD 参与
2. **收益与锁仓挂钩**：锁仓越长，收益率越高，并支持复利滚存
3. **阶梯式提现机制**：引导长期持有，增强系统稳定性
4. **手续费循环利用**：回购销毁、白名单分红、生态建设三轨并行，实现价值闭环
5. **多层激励网络**：直推、平级、团队等级多维度激励，推动生态自增长

### 复利亮点
- 锁定期越长 → 复利越高
- 收益可兑换 GPAD → 兼顾生态币与平台币流动性
- 涡轮 + 冷却池 → 控制流动性、防挤兑

### 核心逻辑
> 运行即治理，参与即建设，价值即激励。
> Operation is Governance, Participation is Construction, Value is Incentive.
